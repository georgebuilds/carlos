Where they converge

The inner loop is the same primitive both projects converge on: stream the model, parse text plus tool calls, dispatch tools, feed results back, terminate when the model stops asking for tools. Both expose the same handful of file / shell / search tools as the coding-agent floor, both keep CLAUDE.md as conversation-start context, both ship a permission gate, and both support subagents that get a fresh context window.

The headline difference

Claude Code is an interactive harness. The loop is tight, the user is in the tight feedback loop, and the architecture is shaped around getting out of the way of one person at a keyboard. Carlos is an event-sourced agent. The loop is identical in shape but every step writes to a durable log, every sub-agent has a lifecycle the supervisor manages, and the surface area is wider: scheduled runs via the daemon, autonomous research with citation grounding, propose-don't-publish skill induction. The cost is more machinery; the payoff is that carlos works when you're not watching.

Neither model is universally better. The inner loop converges because the task does: an LLM with tools, iterating against environmental feedback, until either the model stops asking for tools or a safety rail trips. Everything outside that core is product positioning.
